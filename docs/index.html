How it works

The Android app starts a foreground receiver service, advertises itself on the local Wi-Fi network, and is designed to host a native AirPlay-compatible mirroring engine.

Android Receiver

The APK launches the receiver shell, foreground service, and Android video surface used by the native stack.

Current State

This build is experimental. Discovery, native startup, and Android video decode are wired in, but device compatibility still varies.

Install

  1. Use an Android device running Android 6 or newer.
  2. Tap Download APK and wait for MirrorAir-1.0.1.apk to finish downloading.
  3. If Android blocks the file, allow Install unknown apps for your browser or the Files app.
  4. Open the downloaded file from the Android Files or Downloads app and install it.
  5. After install, open MirrorAir and test the receiver on the same Wi-Fi network as your Mac.

If Install Fails

  1. Delete every older MirrorAir.apk or MirrorAir-1.0.1.apk file from Downloads.
  2. Uninstall any older MirrorAir or MirrorNode app already on the device.
  3. Download the versioned APK again instead of reusing an old browser download.
  4. If Android says There was a problem parsing the package, check that the device is Android 6 or newer.
  5. If Android says App not installed, try installing from the Files app instead of the browser popup.

Current Limits

This build is still experimental. The install package is valid, but Mac to Android mirroring is still under active native integration and may not work on every device yet.
